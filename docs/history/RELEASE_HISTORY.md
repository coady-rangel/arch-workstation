# Release History

This document provides a concise historical overview of the Arch Workstation project's official tagged releases.

Detailed release information is available in [`../roadmap/release_notes/`](../roadmap/release_notes/).

---

## v0.0.9 — Functional Desktop Foundation

Established the initial functional Arch Linux and Hyprland workstation environment.

This milestone represented the project's early desktop foundation before the repository adopted a more formal release and infrastructure model.

---

## v1.0.0 — Continuous Integration Foundation

Established the first stable release milestone and introduced automated shell validation through GitHub Actions.

This release marked the transition from an informal configuration repository toward a validated and maintainable engineering project.

---

## v1.1.0 — Rofi Power Menu

Replaced the previous `wlogout` workflow with a Rofi-based power menu.

This release improved desktop integration, simplified power-management controls, and introduced a dedicated power-menu script.

---

## v1.1.1 — Power Menu Cleanup

Removed obsolete `wlogout` configuration after the migration to the Rofi power menu.

This maintenance release completed the transition introduced in v1.1.0.

---

## v2.0.0 — Deployment Framework and Infrastructure Standardization

Expanded the repository into a reproducible workstation deployment framework.

Major areas of work included:

- Standardized bootstrap, installation, configuration, and validation scripts.
- Declarative package, dependency, and service management.
- Improved installer reliability and preflight validation.
- Shared shell libraries and repository-wide validation.
- Expanded CI validation.
- Completed core desktop functionality and integration.
- Improved infrastructure documentation.

This release represented a major architectural transition from managed dotfiles to workstation infrastructure.

---

## v3.0.0 — Governance Foundation

Established the project's formal governance and engineering framework.

Major areas of work included:

- Project principles and engineering maxims.
- Project constitution and governance documentation.
- Release history and documentation structure.
- Defined roadmap and project-planning practices.
- Improved long-term maintainability and decision consistency.

This release marked the transition from a functional infrastructure repository to a governed workstation platform.
