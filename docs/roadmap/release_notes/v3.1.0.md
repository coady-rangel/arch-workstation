# Lessons Learned

- Release notes should be created before their corresponding Git tags are pushed.
- Documentation should evolve alongside implementation rather than through large retrospective cleanup efforts.
- Git tags, changelog entries, release notes, and GitHub Releases should be treated as one synchronized release system.
- Repository structure should be verified before rewriting navigation documentation.
- Release publication can be automated while retaining deliberate control over version selection and release content.
