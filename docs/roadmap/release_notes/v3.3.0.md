# v3.3.0 — Release Engineering & Validation

**Release Date:** July 23, 2026

# Overview

Version **3.3.0** focuses on improving the project's engineering workflow rather than adding end-user desktop features.

This release introduces a fully automated release pipeline, strengthens repository validation, improves deployment verification, and increases the overall reliability of the development workflow. These improvements establish a solid foundation for the upcoming **v4.0.0 Desktop Completion** milestone.

# New Features

## Automated Release Utility

Added a dedicated release utility located at:

```text
scripts/release.sh
```

The release utility automates common release tasks including:

- Version normalization
- Repository validation
- Release note verification
- README generation
- Repository health checks
- Configuration validation
- Dry-run support
- Release summaries

Example:

```bash
make release VERSION=3.3.0
```

or

```bash
make release VERSION=3.3.0 FLAGS=--dry-run
```

---

## Dry Run Mode

Release automation now supports safe testing without creating Git tags or GitHub releases.

Example:

```bash
make release VERSION=3.3.0 FLAGS=--dry-run
```

Dry runs execute the complete validation pipeline while preventing any permanent changes.

## Release Checklist

The checklist documents the expected release workflow and helps ensure every release follows a consistent process.

# Improvements

## Repository Validation

The release pipeline now validates:

- Clean Git working tree
- No staged changes
- No untracked files
- Required release notes
- Repository health
- README generation
- Configuration deployment
- Required dependencies
- Required system services

These checks significantly reduce the likelihood of incomplete or inconsistent releases.

---

## Configuration Validation

Configuration validation has been expanded to verify deployed configuration symlinks.

Validation now confirms that configuration links resolve to the active repository rather than stale or missing locations, helping detect deployment issues before they affect the workstation.

---

## Release Workflow

The release process now performs a series of automated checks before allowing a release to proceed.

Current workflow:

1. Verify repository status
2. Verify release notes exist
3. Run repository health checks
4. Validate deployed workstation configuration
5. Regenerate project documentation
6. Present a release summary

This ensures release artifacts remain consistent, reproducible, and ready for publication.

---

# Bug Fixes

## Validation Robustness

Resolved an issue where:

```bash
readlink -f
```

could terminate validation when a broken symlink was encountered.

Previously:

- Validation exited immediately.
- Remaining checks were skipped.

Now:

- Broken links are reported individually.
- Validation continues through all configured checks.
- A complete validation summary is generated at the end of execution.

This makes troubleshooting deployment issues significantly easier.

---

## Repository Migration

Improved deployment validation following repository migration.

Configuration links referencing previous repository locations are now detected correctly and can be refreshed using:

```bash
./scripts/deploy-configs.sh
```
